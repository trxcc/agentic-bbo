# Background

Imagine a collaborator wants to benchmark an agent that tunes a data-processing pipeline for a laboratory imaging workflow.
Each evaluation runs a surrogate simulator that approximates image quality and throughput.
The benchmark designer wants a task package that is readable by humans, checkable by code, and directly consumable by an LLM agent.

This directory is not the smallest smoke test; it is a documentation-focused example of how to describe a realistic problem.
