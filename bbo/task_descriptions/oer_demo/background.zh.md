# 背景

`oer_demo` 封装了 BO tutorial 仓库中的析氧反应（OER）催化剂设计案例。
它的源数据表默认直接随工作区提供，位置是 `bbo/tasks/scientific/data/examples/OER/OER.csv`；benchmark 在运行时仍按相对路径 `examples/OER/OER.csv` 使用它。原始数据同时包含催化剂金属身份、组成比例、水热与退火条件、质子浓度、催化剂载量，以及测得的过电位。

这个任务对应的科学决策问题是：在不穷举组合式合成空间的前提下，联合选择组成和工艺条件，使 `10 mA cm^-2` 下的过电位尽可能降低。
这正是贝叶斯优化非常擅长的一类“离散类别 + 连续数值”混合设计问题。

真实部分是缓存后的 OER 数据集，以及从 tutorial 工作流继承来的清洗规则。
模拟部分是 evaluator：它会先清洗表格、对 one-hot 编码做对齐，然后通过拟合的随机森林 oracle 给出分数，而不是真正运行电化学实验。
